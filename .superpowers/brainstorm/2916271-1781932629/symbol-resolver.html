组件 3/4：GDScriptSymbolResolver — 逻辑流分析核心

遍历 AST，构建符号表 + 调用图 + 信号图 + 定义使用链

整体架构

GDScriptSymbolResolver │ ├── Input: ClassNode (AST 根节点) ├── Output: GDScriptAnalysisResult │ │ 遍历 AST, 构建四大数据结构: │ ├── SymbolTable 作用域嵌套符号表 │ ├── global_scope 文件级: 类名, extends, class_name │ ├── class_scope 类级: 成员变量/函数/信号/枚举 │ └── local_scopes[] 函数级: 参数, 局部变量, 嵌套块 │ ├── CallGraph 方法调用关系图 │ └── edges[{caller, callee, site, args[]}] │ ├── SignalGraph 信号连接与发射图 │ └── signals[{name, declaration, emits[], connects[]}] │ └── DefUseChain 变量定义-使用链 └── variables[{name, def_site, reads[], writes[]}]

核心数据结构

SymbolTable — 作用域嵌套
class SymbolTable: │ ├── parent: SymbolTable # 外层作用域 ├── symbols: Dictionary # name → Symbol │ ├── func define(name, kind, node) # 声明 ├── func resolve(name) → Symbol # 查找 (递归向parent) └── func push_scope() → SymbolTable # 创建子作用域
class Symbol: ├── name: String ├── kind: int # CLASS / FUNCTION / VARIABLE / │ SIGNAL / ENUM / PARAMETER / CONSTANT ├── declaration: ASTNode # 指向 AST 中的声明节点 ├── datatype: String # 类型标注 (若有) └── is_exported: bool
作用域示例
extends Node var hp := 100 signal health_changed(old, new)
func take_damage(amount: int): var old_hp = hp hp -= amount if hp <= 0: var msg = "dead" emit("health_changed", old_hp, 0)
━━━━━━━━━━━━━━━━━━━━ class_scope: ├─ hp: VARIABLE (line 2) ├─ health_changed: SIGNAL (line 3) └─ take_damage: FUNCTION (line 5) └─ local_scope: ├─ amount: PARAMETER (line 5) ├─ old_hp: VARIABLE (line 6) └─ if_block_scope: └─ msg: VARIABLE (line 9)

CallGraph — 调用关系

调用模式AST 节点解析方式
self.foo() CallNode(callee=AttributeNode(base=SelfNode, name="foo")) 在类作用域中查找 "foo"
foo() (隐式 self) CallNode(callee=IdentifierNode("foo")) 先在局部作用域查找，再到类作用域
super._ready() CallNode(callee=AttributeNode(base=SuperNode, name="_ready")) 标记为父类调用
obj.method() CallNode(callee=AttributeNode(base=IdentifierNode("obj"), name="method")) 追踪 obj 的类型 → 跨文件查找
node.connect("sig", callable) CallNode(callee=AttributeNode(..., "connect")) 提取信号名 + 回调引用
signal_name.connect(f) (新增) CallNode(callee=AttributeNode(AttributeNode(..., "signal_name"), "connect")) 解析 Signal.connect() 调用

SignalGraph — 信号流程图

class SignalGraph: │ ├── signals: Dictionary # signal_name → SignalInfo │ class SignalInfo: ├── name: String ├── declaration: SignalNode # signal 声明位置 ├── params: Array[String] # 参数名列表 ├── emit_sites: Array[Site] # emit("name") / name.emit() 位置 └── connect_sites: Array[Site] # .connect("name", cb) / name.connect(cb) 位置
class Site: ├── line: int ├── node: ASTNode # 对应的 CallNode ├── enclosing_function: String # 所在函数名 └── arguments: Array # emit 参数 / connect 回调

DefUseChain — 变量读写追踪

写操作 (Define / Write)
  • var x = 1 → 定义 + 写
  • x = 2 → 写
  • x += 1 → 读 + 写
  • func f(x): → 参数定义
读操作 (Read)
  • print(x) → 读
  • y = x + 1 → x 读, y 写
  • if x > 0: → 读
  • return x → 读
Resolve 策略 — 标识符解析优先级:
  1. 当前局部作用域（函数参数 / var 声明）
  2. 外层嵌套块作用域（if/for/while 块内的 var）
  3. 类成员（self.xxx 或隐式 self）
  4. 内置函数（BuiltinUtilityFunction）
  5. 全局单例 / autoload（通过名称匹配）
