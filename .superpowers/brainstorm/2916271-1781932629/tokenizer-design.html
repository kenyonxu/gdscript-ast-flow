组件 1/4：GDScriptTokenizer 设计

词法分析器 — 将 .gd 源码文本转换为 Token 流

Token 数据结构

核心算法：逐字符扫描状态机

func scan() -> GDScriptToken:
    _skip_whitespace()           # 跳过空格/tab

    # 行首检测缩进变化 → 生成 INDENT/DEDENT
    if _at_line_start():
        _check_indent()

    c = _advance()               # 取一个字符

    match c:
        "\n": return NEWLINE
        '"', "'": return _scan_string()
        "0"-"9": return _scan_number()
        "_", "a"-"z", "A"-"Z": return _scan_identifier_or_keyword()
        "#": _skip_comment(); return scan()
        "@": return _scan_annotation()
        "$": return _scan_node_path()
        "=": return _scan_eq_or_assign()  # = 还是 ==
        "!": return _scan_bang()           # ! 还是 !=
        "<": return _scan_less()          # < <= <<
        ">": return _scan_greater()       # > >= >>
        "*": return _scan_star()           # * ** *= **=
        ".": return _scan_period()         # . .. ... .call()
        # ... 其余单字符/双字符运算符

缩进处理（关键难点）

Godot 4.7 做法（参考）
  • 维护 indent_stack: List[int]
  • 每个 NEWLINE 后检查下一行的缩进列号
  • 缩进增加 → 推入栈 + 生成 INDENT
  • 缩进减少 → 弹出栈 + 生成 DEDENT
  • 缩进相同 → 不生成缩进令牌
  • 支持括号内忽略缩进
  • 支持 lambda 独立缩进栈
