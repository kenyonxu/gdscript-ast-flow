纯语法树 vs 语义分析？

确定 AST 节点的"深度"：仅还原语法结构，还是附加类型推断和符号解析

纯语法 AST（当前 3.4 工程的做法）
var hp := 100

TreeVar
├── identifier_id: 3 → "hp"
├── type: null (不推断)
└── init: ExprConstant
　　└── constant_id: 0 → 100

✓ 知道有变量 hp，值为常量 100
✗ 不知道 hp 是 int 类型
✗ 不知道 100 是 int 字面量
语义 AST（Godot 4.7 C++ 解析器的做法）
var hp := 100

VariableNode
├── identifier: "hp"
├── datatype:
│ ├── kind: BUILTIN
│ ├── builtin_type: TYPE_INT
│ └── type_source: INFERRED
└── initializer: LiteralNode
　　└── value: Variant(100, TYPE_INT)

✓ 知道 hp 是 int
✓ 值 100 的类型已知
✓ 类型来源：通过 := 推断
